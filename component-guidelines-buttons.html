Buttons are the control panels of your interface. In a space-themed design system, they are the ignition switches and the communication links that allow users to navigate your digital universe. These button guidelines will put you on the path to a successful design liftoff.

Usage Guidelines

Similar to the Primary Button, the Accent Button serves as a call to action to the user. The best use of the Accent Button is as a confirmation button for the main function of the page. This button should be used sparingly in layouts to prevent overwhelm.

Sizing

Large: For major calls to action, hero sections, or full-width mobile buttons.

Usage Guidelines

Outlined secondary action buttons direct users to alternative options like "Cancel" or "Back", allowing them the option to opt out of selections. The button borders follow a 'Stop Light' system. Red signals that the button action is to Cancel an action. Hovering on Yellow gives a caution of the action. Green indicates the action has been confirmed.

Sizing

Medium: The default size for most forms, cards, and standard UI elements.

Button Best Practices

✓ Do: Use clear, action-oriented verbs for labels (e.g., "Deploy Project", "Save Coordinates").

✗ Don't: Use vague language like "Click Here" or "Submit" if a more descriptive verb is available.

✓ Do: Ensure enough spacing between multiple buttons to prevent accidental clicks.

✗ Don't: Stack multiple Primary buttons next to each other; they will compete for attention.
