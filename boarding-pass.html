UI Audit
TKT-EaMe-336824
1
Header
The airline line reads "FocusRoom Airways" — always uppercase, 10px, letter-spaced. Below it: "BOARDING PASS" in 15px/800 weight. The class badge (Economy, Business, First) floats right.
2
Route Block
Origin planet and destination planet displayed at 28px/800 weight. Departure and arrival times sit below in 12px muted text. The FROM and TO labels are 10px uppercase tracking.
3
Flight Path Indicator
A dashed SVG line with an arrow and a purple destination dot. The label below shows the session duration (e.g. "45 min deep focus"). Centered between origin and destination.
4
Tear Divider
A 1px horizontal rule with two 20px semicircle notches punched from each side — mimicking a physical tear line on a real boarding pass. The notch color matches the page background.
5
Mission Objectives
Task list with a count in the label. Each task preceded by a 6px purple dot. Font: 13px/500 Poppins. Max display: 5 items. Additional items show "+ N more" overflow label.
6
Barcode + Session Token
Procedural SVG barcode. Token format: TKT-{Origin2}{Dest2}-{ID6}. E.g. TKT-EaMe-336824 maps to Earth → Mercury.

Earth is always the departure planet
Always use Earth as the origin. The user always departs from their home planet — Earth. Destinations change; home does not.
✗ Don't

Non-Earth departures break the metaphor
Never set a non-Earth origin. Departure must always be Earth. Having two non-home planets destroys the returning-traveler narrative.

Usage Guidance
When to use
  • Session start screen — shown full-width as the user commits to a focus block
  • Session history feed — compact 340px cards in a vertical list
  • Share / export flows — rendered to image for social sharing
  • Achievement unlocks — boarding pass shown with First Class badge on milestone
Token format

The session token encodes route data: TKT-{Orig2}{Dest2}-{ID6}. First two letters of each planet capitalized. Example: Earth → Saturn = EaSa. Token is globally unique per session.

When not to use
  • As a generic task card outside of timed focus sessions
  • For passive UI states (notifications, settings confirmations)
  • Scaled below 280px width — the route block becomes illegible
  • In dark-mode contexts without switching to the dark variant
Accessibility notes

The light (#FDFDFD) card surface achieves a 13:1 contrast ratio for the dark city name text. The barcode SVG carries an aria-label describing the session token. All interactive states receive focus rings. Barcode is decorative — aria-hidden="true".
